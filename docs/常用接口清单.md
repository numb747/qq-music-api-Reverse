# QQ音乐 web 常用接口清单

> 全部通过 CDP 挂 `JSON.stringify` hook,在 y.qq.com 真实页面上实操捕获**加密前明文 body**得到,非推测。抓取账号 uin=747481755,时间 2026-07-10。

## 0. 调用方式(所有接口通用)

统一入口,POST 到:

```
https://u6.y.qq.com/cgi-bin/musics.fcg?_=<时间戳>&sign=<签名>&encoding=ag-1
```

- body 明文结构:`{ "comm": {...}, "<reqKey>": { "module", "method", "param" } }`,再经 AES-GCM 加密(见 [qqcrypto.js])。
- `sign` = `getSecuritySign(明文body)`。
- 多个接口可放在同一 body 里(`req_1/req_2/...`)一次批量请求。
- 用项目里的 `callQQ(module, method, param)` 单发,或 `callBatch({req_1:{...}})` 批量。

### comm 公共参数(登录态关键)

```json
{
  "cv": 4747474, "ct": 24, "format": "json",
  "inCharset": "utf-8", "outCharset": "utf-8",
  "notice": 0, "platform": "yqq.json", "needNewCode": 1,
  "uin": 747481755,
  "g_tk_new_20200303": 326312414,
  "g_tk": 326312414
}
```

> ⭐ **重要:登录后 `g_tk` 不是 5381**
> - 未登录/匿名:`g_tk = 5381`(默认种子)。搜索等公开接口够用。
> - 已登录:`g_tk = g_tk_new_20200303 = hash33(p_skey)`,本次实测为 `326312414`。
> - `qqclient.js` 里硬编码 `g_tk: 5381`,对需严格校验登录态的接口(收藏/关注/写操作)可能不足,应改成用真实 `p_skey` 计算。hash33 算法见 [[qqmusic-web-login]]。

## 1. 搜索

### 综合/歌曲搜索 `music.search.SearchCgiService / DoSearchForQQMusicDesktop`

```json
{
  "remoteplace": "yqq.yqq.yqq",
```

- `searchid` 前端生成的埋点ID,可给固定长数字串或省略。

## 2. 播放(★核心)

### 获取播放地址(vkey) `music.vkey.GetEVkey / GetUrl`

```json
{
  "guid": "1761485696",
  "songmid": ["000lXvdX0uOg8F"],
  "songtype": [0],
  "uin": "747481755",
  "loginflag": 1,
  "platform": "20",
  "xcdn": 1,
  "filename": ["RS02063RB2qa2HHyA1.mp3"]
}
```

- `songmid` / `songtype` / `filename` 三个数组一一对应,可批量取多首。
- `guid` 客户端标识(纯数字,web 端每次随机生成);`loginflag=1` 带登录态(VIP/付费歌曲必需);`platform` 固定 "20"(web)。

**⚠ filename 不能手工拼(实测纠正)**:`filename` 形如 `RS02063RB2qa2HHyA1.mp3`,前缀(如 `RS02`)标音质,中段是**服务端文件标识**。实测发现该中段(`063RB2qa2HHyA1`)既 ≠ songmid(`000lXvdX0uOg8F`)也 ≠ media_mid(`003Rkghd4UHtr5`),自己拼前缀调 GetEVkey 一律返回空 purl。**正确做法:filename 由 web 前端从歌曲数据里带出,直接照抄接口返回,不要自造。**

**返回结果 → 播放地址拼接(实测)**:

```
完整URL = sip[任一] + purl
sip  = ["http://aqqmusic.tc.qq.com/", "http://sjy6.stream.qqmusic.qq.com/"]
purl = "RS02063RB2qa2HHyA1.mp3?guid=..&vkey=..&uin=..&fromtag=.."
```

- 返回结构:`data.midurlinfo[].purl`(每首一项);无权限时 `purl` 为空串。
- `data.sip` 是 CDN 域名数组,取任一 + `purl` 即可播放。

### 某首歌有哪些音质 `CgiGetTrackInfo` 返回的 `file.size_*`

调 `CgiGetTrackInfo`(见下)后,`data.tracks[].file` 里 `size_*` 字段 >0 表示该音质存在(实测 mid=000lXvdX0uOg8F):

| 字段 | 含义 |
|---|---|
| `size_128mp3` / `size_320mp3` | 128k / 320k MP3 |
| `size_flac` | FLAC 无损 |
| `size_48aac`/`size_96aac`/`size_192aac` | AAC 各码率 |
| `size_96ogg`/`size_192ogg` | OGG 各码率 |
| `size_try` | 试听片段大小 |

> `file` 里还含 `media_mid` 等字段,是前端拼 filename 的原料之一,但完整拼接规则未逆出(见上方 ⚠)。

### 播放页歌词 `music.musichallSong.PlayLyricInfo / GetPlayLyricInfo`

```json
{ "songMID": "000lXvdX0uOg8F", "songID": 102321278 }
```

- `songMID` 必填;`songID`(纯数字)可选,同时传更精准。返回含 lyric/trans/roma。

### 批量歌曲信息 `music.trackInfo.UniformRuleCtrl / CgiGetTrackInfo`

```json
{ "types": [0,0,0,0], "ids": [102321278, 244712794, 215191234, 218659640] }
```

- `ids` 歌曲数字ID数组;`types` 等长的类型数组(全 0 即普通歌曲)。播放列表渲染时批量取歌曲元数据。

## 11. 充值 / 付费(★实测:走腾讯米大师,非自研)

> CDP 实操:右上角「VIP」→ `y.qq.com/portal/vipportal/index.html` → 点「立即开通」。抓到的关键事实如下。**未点最终「确认支付」,不产生真实扣款。**

**核心结论:QQ音乐 web 充值不是自研下单接口,而是拉起腾讯统一支付「米大师(Midas)」收银台。**

- 点「立即开通」后加载收银台 JS:
  ```
  https://midas.gtimg.cn/midas/minipay_v2/jsapi/cashier.js
  ```
  真正的下单/支付在该收银台的 iframe 内完成,属腾讯全平台统一支付体系(QQ钱包/微信/Q币),**不在 `musics.fcg` 加密体系内**,也非 QQ音乐私有,故本清单不展开其内部下单参数。

- 充值页附带的余额/校验接口(普通 HTTP fcg,非加密):

| 接口 | 用途 |
|---|---|
| `c.y.qq.com/share/fcgi-bin/dmrp_activity/fcg_get_user_lebi_num.fcg` | 查用户乐币(乐豆)余额 |
| `music.paycenterapi.LoginStateVerificationApi / GetChargeAccount` (musics.fcg) | 支付账户校验,param `{"appid":"mlive"}` |
| `music.vip.PayAlertSvr / GetPayAlertH5` (musics.fcg) | 付费拦截弹窗,param `{"clientType":"h5","scenes":["yqq_p","yqq_p_u"]}` |
| `userInfo.VipQueryServer / SRFVipQuery_V2` (musics.fcg) | 查会员(VIP/超级会员/绿钻)状态 |

> 若要做「充值」,方向是对接米大师 SDK/H5 收银台(需商户侧参数),而非逆向某个下单接口。数字专辑购买同理走米大师。
